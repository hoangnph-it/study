🔧 SSH Optimization Tips (for High-Latency Servers)

⚡ 4 Ways to Make SSH Faster from HCM City

# Solution What It Does Impact
1 Mosh Mobile Shell — instant local echo over UDP. Typing feels zero-lag even at 300ms. Install with brew install mosh + apt install mosh on server. ★★★★★
2 SSH ControlMaster Reuses existing SSH connections. First connect ~350ms, subsequent sessions instant. Add ControlMaster auto to ~/.ssh/config. ★★★★☆
3 Switch to Singapore Hetzner CPX22 in SG: ~$9.49/mo vs CX22 in DE: ~$4.49/mo. Pay ~$5 more for 10× lower latency (30ms vs 300ms). ★★★★☆
4 tmux / screen Run persistent sessions server-side. Detach/reattach without losing work. Reduces impact of disconnects on high-latency links. ★★★☆☆

📋 Recommended ~/.ssh/config for Hetzner Germany

Host hetzner-de
    HostName your-server-ip
    User root
    # Reuse connections — repeat sessions open instantly
    ControlMaster auto
    ControlPath ~/.ssh/sockets/%r@%h-%p
    ControlPersist 600
    # Compress data over high-latency link
    Compression yes
    # Fastest cipher
    Ciphers aes128-gcm@openssh.com
    # Keep alive
    ServerAliveInterval 30
    ServerAliveCountMax 3

Run mkdir -p ~/.ssh/sockets first. Then connect with ssh hetzner-de or mosh hetzner-de.

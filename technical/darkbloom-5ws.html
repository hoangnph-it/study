What
What is Darkbloom?
A decentralized AI inference network that runs LLM workloads on idle Apple Silicon Macs instead of cloud data centers. It provides an OpenAI-compatible API — just change the base URL and your existing code works (streaming, function calling, all SDKs).
Key idea: 100M+ Apple Silicon Macs sit idle 18+ hours/day. Darkbloom turns them into an AI compute network with end-to-end encryption — operators run your inference but cannot see your data.

Why
Why does it exist?
Problem: AI compute has 3 layers of markup that inflate costs:
NVIDIA (GPU maker)→ Hyperscalers (AWS, GCP, Azure)→ API Providers (OpenAI, etc.)→ End Users pay 3×+ markup
Meanwhile: 100M+ Apple Silicon Macs with powerful ML hardware (273–819 GB/s memory bandwidth, Neural Engines, up to 235B param models) sit idle most of the day.
The Airbnb analogy: Airbnb connected idle rooms to travelers. Uber connected idle cars to riders. Darkbloom connects idle Macs to AI demand. Marginal cost ≈ electricity ($0.01–0.03/hr), so prices undercut centralized providers.

How
How does it work?
4-layer privacy architecture — each independently verifiable:
🔐 Encryption
Requests encrypted on user's device before transmission. Coordinator routes ciphertext. Only target node's hardware-bound key can decrypt.
🛡️ Hardware Verification
Each node holds a key from Apple's Secure Enclave. Attestation chain traces back to Apple's root certificate authority.
🔒 Hardened Runtime
Inference process locked at OS level. Debugger attachment blocked. Memory inspection blocked. Operator cannot extract data.
✍️ Traceable Output
Every response signed by the specific machine. Full attestation chain published. Anyone can verify independently.
